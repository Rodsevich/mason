# `ai_codegen_example`

Reference brick that exercises every major facet of masonex's `| ai` filter:

- Plain literal prompt with `expect: line`.
- Mustache substitution inside a literal prompt (`{{className}}` resolved before being sent).
- Nested `| ai` calls — a tag whose prompt is itself a string built with another tag.
- `expect: identifier` with `case: kebab`.
- `max_chars` validation.

## Run with mock fixtures (no network)

```sh
masonex make ai_codegen_example \
  --no-ai=false  # default
  -- --use-mock  # exposed by the runner when available
```

The orchestrator reads `brick_test/ai_fixtures.yaml` and replays canned
responses. Used by CI and by `dart test`.

## Run with a real provider

1. Configure once: `masonex provider setup`
2. Render: `masonex make ai_codegen_example`

Cache lives under `<projectRoot>/.masonex/cache/ai/`. Inspect with
`masonex ai-trace` and `masonex ai-cache stats`.

## What the brick generates

A single `lib/<class_name>.dart` file with:

- A 1-line dartdoc generated by AI.
- A multilingual tagline (the prompt asks the model to compose itself a
  language-specific string by chaining two `| ai` calls).
- A short marketing line with `max_chars` cap.
- A kebab-case identifier post-processed via the `case` parameter.

See `__brick__/lib/{{className.snakeCase()}}.dart` for the templates.
